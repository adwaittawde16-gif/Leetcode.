## 👨‍💻 About This Repository

This repo documents my ongoing journey of solving LeetCode problems to sharpen my problem-solving skills, prepare for technical interviews, and deepen my understanding of data structures and algorithms.

Every solution here represents not just an answer, but a lesson learned — whether it was about thinking recursively, optimizing from O(n²) to O(n log n), or simply learning to read the problem more carefully.

**Languages used:** Java · C

---

## 🗺️ My Learning Journey

I started this repository to hold myself accountable and track real progress over time. LeetCode can feel overwhelming at first — there are thousands of problems and it's easy to feel lost. My approach:

1. **Start with Easy** — Build confidence and internalize core patterns.
2. **Move to Medium** — Where most interview questions live. Pattern recognition becomes key.
3. **Tackle Hard** — Less about memorization, more about creative thinking under pressure.

Along the way, I've developed a stronger intuition for:
- When to use a HashMap vs. a two-pointer approach
- Recognizing sliding window and divide-and-conquer patterns
- Writing clean, readable code even under time pressure
